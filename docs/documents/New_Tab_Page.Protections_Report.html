    Setup

    • Widget ID: "protections"
    • Add it to the widgets + widgetConfigs fields on initialSetup
    • Example:
    {
      "...": "...",
      "widgets": [
        {"...":  "..."},
        {"id": "protections"}
      ],
      "widgetConfigs": [
        {"...":  "..."},
        {"id": "protections", "visibility": "visible" }
      ]
    }

    Requests:

    protections_getData

    • "NewTab Messages".ProtectionsGetDataRequest
    • Used to fetch the initial data (during the first render)
    • returns "NewTab Messages".ProtectionsData
    {
       "totalCount": 84,
       "totalCookiePopUpsBlocked": 23
    }

    protections_getConfig

    • "NewTab Messages".ProtectionsGetDataRequest
    • Used to fetch the initial config data (eg: expanded vs collapsed)
    • returns "NewTab Messages".ProtectionsConfig
    {
       "expansion": "collapsed",
       "feed": "privacy-stats",
       "showBurnAnimation": true,
       "showProtectionsReportNewLabel": true
    }

    Subscriptions:

    protections_onDataUpdate

    • "NewTab Messages".ProtectionsOnDataUpdateSubscription.
    • The tracker/company data used in the feed.
    • returns "NewTab Messages".ProtectionsData

    protections_onConfigUpdate

    • "NewTab Messages".ProtectionsOnConfigUpdateSubscription.
    • The widget config
    • returns "NewTab Messages".ProtectionsConfig

    Notifications:

    protections_setConfig

    • "NewTab Messages".ProtectionsSetConfigNotification
    • Sent when the user toggles the expansion of the stats
    • sends "NewTab Messages".ProtectionsConfig
    • example payload:
    {
       "expansion": "collapsed",
       "feed": "privacy-stats",
       "showBurnAnimation": true
    }
